# Terminal Core 009-010-012 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the MVP2-008 transcript-only surface with a live SwiftTerm-backed terminal runtime that supports PTY lifecycle, split/focus/resize deck behavior, and baseline terminal UX for copy/paste/find/scrollback/hyperlink.

**Architecture:** Keep PTY lifecycle and the authoritative live-session registry in Rust (`hc-runtime`) and keep SwiftTerm as the renderer/input surface inside the macOS app. Use `hc-ffi` as a thin transport over Rust-owned session commands/queries/output drain; keep pane layout, first-responder handling, and focus UI in SwiftUI/AppKit. Because the controlling backlog still binds `MVP2-009` to `AC-14`, include a persisted terminal-session restore slice in this plan: serializable launch descriptor + terminal geometry + cold relaunch proof for terminal sessions after app restart, while broader project/task/layout restore remains in `MVP2-046/047`.

**Tech Stack:** Rust 2024, `portable-pty`, C ABI FFI, SwiftUI/AppKit, SwiftTerm 1.12.0, Swift Testing, Cargo tests.

---

## Scope Locks

- `MVP2-009` closes live PTY lifecycle plus the terminal-session restore slice needed for its current `AC-14` attachment: serializable launch descriptor, terminal geometry, and cold relaunch proof for terminal sessions. Full project/task/layout restore breadth remains follow-up work in `MVP2-046/047`.
- `MVP2-010` closes split/focus/resize inside the central `WF-02` terminal deck, not tabs, multi-window, or detached panes. It must preserve future `Session Stack` and `Inspector` seams and follow `docs/theme/UI_THEME.md` constraints for no-line hierarchy, tonal separation, terminal dominance, and keyboard-first focus.
- `MVP2-012` should reuse SwiftTerm built-ins where available instead of rebuilding terminal UX in SwiftUI, but verification must explicitly cover scrollback after live PTY output and resize.
- Rust core remains the source of truth for live-session lifecycle and queryable session state; `hc-ffi` and Swift UI must not create an alternate session registry.
- Control-plane snapshot, task linkage, shell integration, and session metadata surfaces remain follow-up tasks (`MVP2-011`, `MVP2-014`, `MVP2-016`, `MVP2-046`, `MVP2-047`).

## Governing UI Guardrails For `WF-02`

- Keep the split deck inside the central terminal surface; do not consume the reserved seams for `Session Stack` (`220~280px`) or `Inspector` (`320~400px`) defined by the wireframe spec.
- Follow the `No-line rule` and `Tonal separation` from `docs/theme/UI_THEME.md`: default hierarchy should come from surface tone, padding, and spacing, not boxy card borders or heavy dividers.
- Terminal focus remains the default. Pane focus, split actions, and resize flows must be keyboard-first, and supporting controls must not dominate the terminal surface.
- Failure / degraded / empty copy in the deck must stay cause-first and next-action oriented, using the shared operational vocabulary from the PRD and UI theme docs.

- A callback-heavy FFI design will slow implementation and increase unsafe surface area. Prefer drain-based binary reads first.

- Split layout can become over-engineered. Keep the first tree model focused on pane split/collapse/focus/ratio only.
