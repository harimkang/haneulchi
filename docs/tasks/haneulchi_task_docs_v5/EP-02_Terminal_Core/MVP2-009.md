## 2. Objective
이 task는 **PTY session spawn / kill / resize / restore lifecycle 구현** 를 실제 구현 단위로 분해해, Terminal Core의 결과를 제품 화면과 수용 기준에 연결한다.

## 3. Why This Task Exists
- 이 task는 **Terminal Core** 의 outcome을 실제 제품 기능으로 내린다.
- 관련 사용자는 UC-03 — daily terminal navigation / UC-05 — split/focus terminal deck 흐름을 통해 이 기능을 직접 체감한다.
- 완료되면 `WF-02` 화면군에서 operator-visible 동작이 닫혀야 한다.

## 4. In Scope
- `PTY session spawn / kill / resize / restore lifecycle 구현` 에 직접 필요한 UI, 상태, 액션, 데이터 연결
- `Project Focus / Terminal Deck` 화면에서 사용자 흐름이 닫히도록 만드는 최소 구현
- `AC-03, AC-14` 또는 해당 epic gate에 필요한 검증 훅
- state/event/projection/diagnostics 정합성 유지

## 5. Out of Scope
- 별도 P1/P2로 밀린 확장 항목
- unrelated screen redesign
- raw terminal parsing에 의존한 state 추정 확대
- full browser / multi-window / external sync 같은 비핵심 확장

## 6. Implementation Breakdown
- T1. PTY session spawn / kill / resize / restore lifecycle 구현 범위를 domain/state/API 단위로 분리
- T2. Project Focus / Terminal Deck 화면에서 필요한 view-model / interaction contract 연결
- T3. persistence / event / projection 업데이트
- T4. failure / degraded / empty 상태 정의
- T5. demo 증적과 regression 체크 정리


## 7. Data / Interface Notes
- 주요 모듈 후보: runtime-core lane

- screen contract는 stable vocabulary(manual, claimed, retry_due, workflow_ok 등)를 재사용한다.
